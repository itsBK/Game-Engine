Introduction

A lightweight, modular game engine built for 2D and 3D indie game development.

🎯

Cross-Platform

Linux-first development with Windows support planned. Built with portability in mind.

⚡

ECS Architecture

Data-oriented Entity Component System for efficient game object management.

🎨

Modern Graphics

OpenGL 4.x rendering with optional Vulkan abstraction layer.

📦

Modular Design

Clean separation of concerns with minimal dependencies.

Engine Overview

Architecture Overview

Layer 1: Application

Game logic, scenes, scripts, gameplay systems

Layer 2: Core Systems

ECS, Physics, Scene Graph, Audio

Layer 3: Rendering

Renderer, Shaders, Materials, Cameras

Layer 4: Platform

Window, Input, File System, Logging

Layer 5: Math & Utilities

Vector/Matrix math, containers, utilities

Design Principle: Each layer only depends on layers below it. Higher-level systems should never directly access lower-level details.

ECS Core

Entity Component System architecture for efficient game object management.

ECS Concepts

Entity

Just an ID. A unique identifier that represents a game object.

Entity = uint32_t

Component

Raw data. No logic, just data structures attached to entities.

struct Position { float x, y, z; }

System

Logic. Functions that operate on entities with specific components.

MovementSystem, RenderSystem

System Implementation Pattern

// MovementSystem example
class MovementSystem : public System {
public:
    void init() override {
        m_requiredComponents = {
            ComponentType::Transform,
            ComponentType::RigidBody
        };
    }
    
    void update(float dt) override {
        for (Entity entity : getEntities()) {
            auto& transform = m_world->getComponent<Transform>(entity);
            auto& body = m_world->getComponent<RigidBody>(entity);
            
            transform.position += body.velocity * dt;
            transform.rotation += body.angularVelocity * dt;
        }
    }
};

Rigid Bodies

Physical simulation of rigid body dynamics.

Body Types

Static Bodies

Infinite mass, zero velocity. Used for ground, walls.

mass = INFINITY
velocity = Vec3(0)
invMass = 0

Dynamic Bodies

Finite mass, simulated physics. Used for players, crates.

mass = 1.0f
velocity = computed
invMass = 1.0f

Integration

// Semi-implicit Euler integration
void RigidBody::integrate(float dt) {
    // Apply gravity
    velocity += gravity * dt;
    
    // Apply damping
    velocity *= (1.0f - linearDamping);
    angularVelocity *= (1.0f - angularDamping);
    
    // Integrate position
    position += velocity * dt;
    rotation += angularVelocity * dt;
    
    // Reset forces
    force = Vec3(0);
    torque = Vec3(0);
}

// TRS Matrix

Vec3 worldPos = transform * localPos;

// Decomposition

class IAssetLoader {
public:
    virtual ~IAssetLoader() = default;
    virtual bool canLoad(const std::string& extension) = 0;

class TextureLoader : public IAssetLoader {
public:

        return ext == ".png" || ext == ".jpg";
